# 02 · 현재 구조

## 모듈 맵
```
src/
├── data/
│   ├── schedule.js   → PERIODS, DAYS, TEACHERS (교시·요일·선생님 기본 시간표)
│   └── floors.js     → FLOORS, OFFICE_IDS, ADMIN_PIN (층별 평면도 좌표)
└── lib/
    ├── time.js       → toMins, getNowMins, getCurrentPeriodIndex,
    │                    getNextPeriodIndex, getBreakAfterIndex, getTodayIndex
    ├── location.js   → 시간표 조회·위치 계산·localStorage 헬퍼

```

## 데이터 계약 (코드에서 확정된 사실)

### 시간표 배열
`TEACHERS[].schedule = schedule[dayIdx][periodIdx]`
- `dayIdx`: 0=월 ~ 4=금 (5행). 주말은 `getTodayIndex()`가 월(0)로 고정.
- `periodIdx`: 0~7 (8열). PERIODS 순서 = 1·2·3·4교시 / 점심(idx 4) / 5·6·7교시.
- 값: `'과목(학년-반)'` 문자열 (예: `'수학(2-1)'`) 또는 `null`.
- 점심(idx 4)은 보통 `null`.
- `parseClassLabel()`이 문자열 끝에서만 `(숫자-숫자)` 패턴을 인식해 `{subject, grade, class, label}`로 정규화 —
  과목명 중간에 괄호가 있어도(`'국어(문학)'`) 오파싱되지 않음.

### 임시 일정 (override)
- localStorage 키: `teacher_overrides` (배열)
- 객체 형태:
  ```js
  { teacherId, date: 'YYYY-MM-DD', periodIdx, label, room, floor, note }
  ```
- `room`: room id (예: `'2-1'`, 또는 교무실은 `'office'`)
- `floor`: 층 번호(정수) 또는 null
- `label`: 화면 표시 텍스트 (예: `'2-1'`, `'상담실'`)
- `getTeacherLocation()`이 오늘 날짜 + 교시가 일치하는 override를 최우선 적용.
- 헬퍼: `loadOverrides()`, `saveOverrides(list)` (`location.js`)

### AI 분석 시간표
- localStorage 키: `teacher_schedules`
- 형태: `{ [teacherId]: { schedule, updatedAt, source: 'image_ai' } }`
- `schedule`은 위 시간표 배열(5×8)과 동일 형태.
- 헬퍼: `saveAiSchedule(teacherId, schedule)`, `loadAiSchedules()` (`location.js`)

### 기타 상수
- `ADMIN_PIN` (`floors.js`) — 클라이언트에 하드코딩(값은 소스 참조). Phase C에서 서버 인증으로 대체 예정.
- `OFFICE_IDS`: 층 번호 → 그 층 교무실 room id 매핑 (`floors.js`, 5·4·3층만 존재)
- `FLOORS[n] = { W, H, label, rooms: [{id,label,x,y,w,h,type,...}] }` — 좌표 데이터는 이 파일 한 곳에만 존재(단일 소스).

## 위치 계산 핵심 함수 (`location.js`)
- `getTeacherLocation(teacherId, dayIdx, periodIdx)` → `{type,label,room,floor,note?}`
  (우선순위: 임시일정 > AI시간표 > 기본시간표; 점심·쉬는시간 특수 처리)
- `getEffectiveSchedule(teacherId)` → AI시간표 있으면 그것, 없으면 기본
- `buildTimeline(teacherId)` → 오늘 교시별 위치 + 상태(past/now/next/future)
- `getNextMove(teacherId)` → 현재→다음 위치 이동 정보
- `resolveRoom(roomId, locFloor, currentFloor, floorData)` → room id를 실제 room 객체로

## 지도 렌더링 (`map.js`)
- `_computePinPositions()`가 같은 방의 여러 선생님 핀을 원형으로 분산 배치하고,
  `_drawAllPins()`(렌더)와 `hitTestPin()`(클릭 판정)이 이 좌표를 공유 — 겹친 핀도 정확히 클릭 판정됨.
- `_drawTeacherRoute()` + `_drawRouteArrow()` — 선택된 선생님의 현재 방을 강조하고,
  다음 교시 위치가 같은 층의 다른 방이면 점선 화살표로 동선 표시.
