# 03 · 개발 로드맵

우선순위: **Phase A(MVP 완성)** → Phase B(안정화) → Phase C(확장).

---

## Phase A — MVP 3화면 완성  ✅ (이번 세션 대상)

| 작업 | 수용 기준 |
|------|-----------|
| `index.html` 학생 화면 | 선생님 선택 시 층 전환 + 방 줌인, 현재/다음 위치·타임라인 표시 |
| `admin.html` 관리자 | PIN 1234 통과 → 임시일정 등록/삭제 → index에 즉시 반영(override 우선) |
| `upload.html` 업로드 | 사진 → Claude 분석 → 편집 표 → `saveAiSchedule` 저장 → index 반영 |
| 공통 네비게이션 | 세 화면 상호 이동 링크 |

## Phase B — 안정화 (README 버그 정리)

| 작업 | 수용 기준 |
|------|-----------|
| 동선 화살표 완성 (`map.js` `_drawTeacherRoute`) | 현재→다음 위치가 같은 층일 때 화살표 렌더 |
| 히트테스트 개선 (`map.js` `hitTestPin`) | 같은 방 다수 핀도 분산 좌표 기준 개별 클릭 판정 |
| 시간표 파싱 정규화 | `'과목(학년-반)'` → `{subject, grade, class}` 객체로 정규화, 괄호 오파싱 방지 |
| 주말 처리 주석/정책 | `getTodayIndex()` 주말→월 고정 의도 명시, 필요 시 "주말 안내" UI |
| FLOORS 단일 소스 확인 | 좌표 데이터가 `floors.js` 한 곳에만 존재하도록 유지 |

## Phase C — 확장

| 작업 | 수용 기준 |
|------|-----------|
| 기기 간 동기화 (Firebase Firestore / Supabase) | A 기기에서 등록한 임시일정이 전자칠판에 자동 반영 |
| 서버사이드 인증 / 개별 PIN / 구글 로그인 | 클라이언트 PIN 노출 제거, 선생님별 계정 |
| Claude API 서버 프록시 | API 키를 브라우저에서 제거(현재 upload.html은 키 노출 위험) |
| 실제 학교 시간표 데이터 입력 | 샘플 6명 → 실제 교사·시간표로 교체 |
| 반응형 / 전자칠판 풀스크린 최적화 | 대형 화면·터치 환경 UI |
